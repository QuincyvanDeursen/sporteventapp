Project SportEventApp Omschrijving

Deze website is gemaakt door Quincy van Deursen (2113709) voor het project client side web programming. Het framework Angular is gebruikt om deze front-end op te zetten. De back-end is opgezet met NestJS. De databases die gebruikt worden zijn MongoDB en Neo4J. Voor MongoDB wordt ook Mongoose gebruikt. Het project in zijn geheel staat in een nx workspace. De applicatie is getest aan de hand van unit, integratie en end2end testen. Voor de testen wordt ook gebruik gemaakt van Jest.

De casus van deze webapplicatie is als volgt: Sportclubs bieden vaak evenementen aan zoals introductielessen, workshops, lezingen, wedstrijden etc. Het probleem daarbij is echter dat je vaak de betreffende sportclub nauw moet volgen om van de evenementen af te weten. Deze applicatie is bedacht om daar verandering in te brengen. Het idee is dat sportscholen de door hun georganiseerde evenementen aanbieden op de applicatie. Gebruikers kunnen deze evenementen zien en de details er van bekijken. Dat maakt het makkelijk om te zien welke evenementen er gehouden worden. Het idee is dat gebruikers kunnen filteren op locatie en soort evenement. Ook kunnen gebruikers vrienden volgen. De gebruiker kan inzien waarvoor de vrienden zich hebben ingeschreven. Ook wordt de gebruiker eventuele evenement aanbevelingen getoond op basis vriend-inschrijvingen of inschrijf geschiedenis.

Hoewel de applicatie gebruikt maakt van NoSQL, maakt het onderstaande ERD wel duidelijk welke relaties de entiteiten met elkaar hebben.

  • Een user kan zich inschrijven voor één of meerdere events.
  • Een event kan meerdere inschrijvingen hebben.
  • Een event wordt gehouden bij één sportclub.
  • Een sportclub kan meerdere events hosten.
  • Een medewerker-gebruiker werkt bij een club.
  • Een medewerker-gebruiker kan evenementen aanmaken.

User Stories

    Klant-gebruiker

  • Als gebruiker wil ik kunnen registeren.
  • Als gebruiker wil ik kunnen Inloggen.
  • Als gebruiker wil ik een overzicht van evenementen kunnen bekijken.
  • Als gebruiker wil ik de details van een evenement kunnen bekijken.
  • Als gebruiker wil ik kunnen filteren op locatie en soort evenement.
  • Als gebruiker wil ik kunnen inschrijven voor een evenement.
  • Als gebruiker wil ik vrienden kunnen volgen.
  • Als gebruiker wil ik kunnen zien waarvoor mijn vrienden zich hebben ingeschreven.
  • Als gebruiker wil ik suggesties krijgen voor evenementen op basis van vriend-inschrijvingen/eerdere inschrijvingen.

    SportClub-gebruiker

  • Als gebruiker wil ik kunnen registeren.
  • Als gebruiker wil ik kunnen Inloggen.
  • Als gebruiker wil ik een overzicht van evenementen kunnen bekijken van mijn sportclub.
  • Als gebruiker wil ik een evenement kunnen aanpassen/verwijderen.
  • Als gebruiker wil ik evenementen kunnen aanbieden voor de sportschool waarvoor ik werk.
  • Als gebruiker wil ik de potentiële deelnemers van een evenement kunnen zien.
  • Als gebruiker wil ik de potentiële opbrengst van een evenement kunnen zien.
